# Requirements Document: Редизайн публичной страницы мастера

## Introduction

Редизайн публичной страницы мастера с современным glass morphism дизайном, добавлением портфолио и улучшенным UX для онлайн-записи. Новый дизайн должен увеличить конверсию в записи и создать wow-эффект для клиентов мастеров.

## Glossary

- **Master**: Пользователь системы (мастер/специалист), который предоставляет услуги
- **Client**: Клиент мастера, который записывается на услугу через публичную страницу
- **Public_Page**: Публичная страница мастера, доступная по ссылке `/m/{slug}`
- **Portfolio**: Галерея работ мастера с изображениями
- **Glass_Morphism**: Стиль дизайна с полупрозрачными элементами и эффектом размытия фона
- **Booking_Modal**: Модальное окно для онлайн-записи с многошаговой формой
- **Service**: Услуга, которую предоставляет мастер
- **Slug**: Уникальный идентификатор мастера в URL (например, `ivan-petrov`)

## Requirements

### Requirement 1: Современный дизайн с Glass Morphism

**User Story:** Как клиент мастера, я хочу видеть современную и привлекательную публичную страницу, чтобы у меня возникло доверие к мастеру и желание записаться.

#### Acceptance Criteria

1. WHEN клиент открывает публичную страницу, THE System SHALL отобразить дизайн с glass morphism эффектами (полупрозрачные карточки с размытием)
2. WHEN страница загружается, THE System SHALL применить плавные анимации появления элементов (fade-in, slide-in)
3. WHEN клиент наводит курсор на интерактивные элементы, THE System SHALL показать hover эффекты с плавными переходами
4. THE System SHALL использовать фиксированный header с аватаром мастера и кнопкой "Поделиться"
5. THE System SHALL отображать плавающую панель записи внизу экрана с кнопками "Записаться" и "Позвонить"
6. THE System SHALL применять адаптивный дизайн для корректного отображения на всех устройствах

### Requirement 2: Карточка профиля мастера

**User Story:** Как клиент мастера, я хочу видеть полную информацию о мастере в одном месте, чтобы быстро понять, кто он и чем занимается.

#### Acceptance Criteria

1. THE System SHALL отображать карточку профиля с большим аватаром мастера (128x128px) в скругленном контейнере
2. THE System SHALL показывать имя мастера крупным шрифтом (text-4xl, font-black)
3. THE System SHALL отображать роль/специализацию мастера заглавными буквами с увеличенным межбуквенным расстоянием
4. THE System SHALL показывать локацию мастера с иконкой MapPin
5. WHEN мастер заполнил биографию, THE System SHALL отображать её в отдельной секции под карточкой профиля
6. THE System SHALL применять анимацию плавающего движения (animate-float) к карточке профиля

### Requirement 3: Социальные сети и контакты

**User Story:** Как клиент мастера, я хочу иметь возможность связаться с мастером через удобный для меня канал, чтобы задать вопросы или записаться.

#### Acceptance Criteria

1. THE System SHALL отображать сетку из 4 иконок социальных сетей (Telegram, Instagram, VK, WhatsApp)
2. WHEN клиент нажимает на иконку социальной сети, THE System SHALL открыть профиль мастера в новой вкладке
3. WHEN мастер не указал социальную сеть, THE System SHALL скрыть соответствующую иконку
4. THE System SHALL отображать карточку с номером телефона и кнопкой прямого звонка
5. WHEN клиент нажимает на карточку телефона, THE System SHALL инициировать звонок через tel: протокол
6. THE System SHALL применять hover эффекты к иконкам социальных сетей (масштабирование и поворот)

### Requirement 4: Список услуг

**User Story:** Как клиент мастера, я хочу видеть все доступные услуги с ценами и длительностью, чтобы выбрать подходящую и записаться.

#### Acceptance Criteria

1. THE System SHALL отображать список всех активных услуг мастера в виде карточек
2. WHEN клиент просматривает услугу, THE System SHALL показать название, длительность в минутах и цену в рублях
3. WHEN клиент нажимает на карточку услуги, THE System SHALL открыть модальное окно записи с предвыбранной услугой
4. THE System SHALL применять hover эффект к карточкам услуг (изменение фона и иконки)
5. THE System SHALL отображать иконку Clock рядом с длительностью услуги
6. THE System SHALL сортировать услуги по порядку, установленному мастером

### Requirement 5: Портфолио мастера

**User Story:** Как клиент мастера, я хочу видеть примеры работ мастера, чтобы оценить качество и стиль его работы перед записью.

#### Acceptance Criteria

1. THE System SHALL отображать портфолио в виде сетки 2 колонки с соотношением сторон 4:5
2. WHEN клиент наводит курсор на изображение, THE System SHALL применить эффект масштабирования (scale-110) и показать название работы
3. WHEN мастер не загрузил портфолио, THE System SHALL скрыть секцию портфолио
4. THE System SHALL отображать только видимые (is_visible = true) работы портфолио
5. THE System SHALL сортировать работы по полю sort_order
6. THE System SHALL применять скругленные углы (rounded-[32px]) к карточкам портфолио

### Requirement 6: Модальное окно записи

**User Story:** Как клиент мастера, я хочу пройти простой и понятный процесс записи, чтобы быстро забронировать время без лишних сложностей.

#### Acceptance Criteria

1. WHEN клиент нажимает "Записаться", THE System SHALL открыть модальное окно с многошаговой формой
2. THE System SHALL отображать прогресс-бар вверху модального окна с текущим шагом
3. THE System SHALL реализовать 5 шагов: выбор услуги → выбор даты → выбор времени → ввод контактов → успех
4. WHEN клиент выбирает услугу из карточки, THE System SHALL пропустить шаг выбора услуги и сразу показать выбор даты
5. WHEN клиент переходит между шагами, THE System SHALL применять плавные анимации (slide-in, fade-in)
6. WHEN клиент завершает запись, THE System SHALL показать экран успеха с деталями записи
7. THE System SHALL сохранять данные между шагами и позволять вернуться назад
8. WHEN клиент закрывает модальное окно, THE System SHALL сбросить все данные формы

### Requirement 7: Управление портфолио в админке

**User Story:** Как мастер, я хочу загружать и управлять своим портфолио, чтобы показывать клиентам примеры своих работ.

#### Acceptance Criteria

1. THE System SHALL предоставить страницу управления портфолио в админке мастера
2. WHEN мастер загружает изображение, THE System SHALL создать оригинал (макс 2000x2000px) и thumbnail (400x500px)
3. WHEN мастер загружает изображение, THE System SHALL оптимизировать его размер без потери качества
4. THE System SHALL поддерживать drag & drop загрузку множественных файлов
5. THE System SHALL позволять мастеру изменять порядок работ через drag & drop
6. THE System SHALL позволять мастеру добавлять название и описание к каждой работе
7. THE System SHALL позволять мастеру скрывать/показывать работы (is_visible)
8. THE System SHALL применять лимиты по тарифу (Базовая: 5, Профессиональная: 30, Максимальная: безлимит)
9. WHEN мастер достигает лимита, THE System SHALL показать предупреждение и предложить обновить тариф
10. THE System SHALL поддерживать форматы изображений: JPG, PNG, WebP

### Requirement 8: Настройки публичной страницы

**User Story:** Как мастер, я хочу настраивать внешний вид своей публичной страницы, чтобы она отражала мой стиль и бренд.

#### Acceptance Criteria

1. THE System SHALL предоставить страницу настроек публичной страницы в админке
2. THE System SHALL позволять мастеру редактировать заголовок страницы (site_title)
3. THE System SHALL позволять мастеру редактировать описание для SEO (site_description)
4. THE System SHALL позволять мастеру добавлять биографию (site_bio)
5. THE System SHALL позволять мастеру указывать локацию (site_location)
6. THE System SHALL позволять мастеру добавлять ссылки на социальные сети (Instagram, VK, Telegram, WhatsApp)
7. THE System SHALL позволять мастеру выбирать основной цвет темы (theme_color) через color picker
8. THE System SHALL позволять мастеру настраивать градиент фона (site_gradient_from, site_gradient_to)
9. THE System SHALL показывать живой превью страницы при изменении настроек
10. WHEN мастер сохраняет настройки, THE System SHALL обновить публичную страницу в реальном времени

### Requirement 9: SEO оптимизация

**User Story:** Как мастер, я хочу, чтобы моя публичная страница хорошо индексировалась поисковыми системами, чтобы клиенты могли найти меня через поиск.

#### Acceptance Criteria

1. THE System SHALL генерировать метатеги title и description на основе настроек мастера
2. THE System SHALL добавлять Open Graph метатеги для корректного отображения в социальных сетях
3. THE System SHALL использовать аватар мастера как og:image
4. THE System SHALL генерировать JSON-LD микроразметку schema.org типа LocalBusiness
5. THE System SHALL включать в микроразметку: name, description, image, telephone, address
6. THE System SHALL использовать семантические HTML теги (header, main, section, footer)

### Requirement 10: Адаптивность и производительность

**User Story:** Как клиент мастера, я хочу, чтобы публичная страница быстро загружалась и корректно работала на моем устройстве, независимо от его типа.

#### Acceptance Criteria

1. THE System SHALL обеспечить корректное отображение на экранах от 320px до 2560px
2. THE System SHALL оптимизировать изображения портфолио для быстрой загрузки
3. THE System SHALL использовать lazy loading для изображений портфолио
4. THE System SHALL кэшировать публичную страницу на 5 минут
5. THE System SHALL обеспечить время загрузки страницы менее 2 секунд
6. THE System SHALL применять responsive дизайн для всех компонентов
7. WHEN клиент использует мобильное устройство, THE System SHALL адаптировать размеры шрифтов и отступов

### Requirement 11: Кнопка "Поделиться"

**User Story:** Как клиент мастера, я хочу иметь возможность поделиться страницей мастера с друзьями, чтобы порекомендовать его услуги.

#### Acceptance Criteria

1. THE System SHALL отображать кнопку "Поделиться" в фиксированном header
2. WHEN клиент нажимает кнопку "Поделиться", THE System SHALL открыть нативное меню шаринга (Web Share API)
3. WHEN Web Share API не поддерживается, THE System SHALL скопировать ссылку в буфер обмена и показать уведомление
4. THE System SHALL передавать в шаринг: title, text, url публичной страницы
5. THE System SHALL применять hover эффект к кнопке "Поделиться"

### Requirement 12: Интеграция с существующей системой записи

**User Story:** Как разработчик, я хочу сохранить существующую логику записи, чтобы не нарушить работающий функционал.

#### Acceptance Criteria

1. THE System SHALL использовать существующий endpoint `/api/public/booking/{slug}/slots` для получения доступных слотов
2. THE System SHALL использовать существующий endpoint `/api/public/booking/{slug}/book` для создания записи
3. THE System SHALL сохранять существующую логику проверки доступности времени
4. THE System SHALL сохранять существующую логику создания/поиска клиента
5. THE System SHALL применять новый UI к модальному окну записи без изменения backend логики
6. THE System SHALL отправлять те же данные в API, что и старая версия
