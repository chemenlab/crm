# Spec Summary: Onboarding System Improvements

## Цель

Улучшить существующую систему онбординга MasterPlan CRM путем добавления контекстных подсказок на все страницы, автоматического отслеживания прогресса и создания базы знаний.

## Текущее состояние

**Готовность: 60%**

✅ **Реализовано:**
- Backend API для прогресса онбординга
- Интерактивный тур (driver.js)
- Компонент Hint для контекстных подсказок
- Прогресс-бар и приветственная модалка
- Подсказки на страницах: Dashboard, Services, Clients, Portfolio, Settings/Schedule

❌ **Не реализовано:**
- Подсказки на страницах: Calendar, Finance, Settings/Profile, Settings/Notifications
- Автоматическое отслеживание прогресса в контроллерах
- Toast уведомления о достижениях
- База знаний (опционально)
- Статистика для администратора (опционально)

## Ключевые требования

### 1. Контекстные подсказки (Requirements 1, 3, 6)
- Добавить подсказки на все основные страницы
- Условное отображение (только когда актуально)
- Автоматическое скрытие после просмотра
- 3 типа: info, tip, warning

### 2. Автоматическое отслеживание (Requirements 2, 5)
- Отслеживание в ServiceController (first_service)
- Отслеживание в ClientController (first_client)
- Отслеживание в AppointmentController (first_appointment)
- Отслеживание в SettingsController (schedule_setup, public_page_setup, notification_setup)
- Обработка ошибок без прерывания основного процесса

### 3. Toast уведомления (Requirement 8)
- Уведомления при выполнении каждого шага
- Поздравительная модалка при завершении всех шагов
- Автоматическое скрытие через 5 секунд
- Использование shadcn/ui toast

### 4. Улучшенный прогресс-бар (Requirement 7)
- Детальный список шагов с галочками
- Процент выполнения
- Кнопка "Начать заново"
- Автоматическое скрытие после завершения

### 5. База знаний (Requirement 4, опционально)
- Страница /app/help
- Категории статей
- Поиск по статьям
- Рендеринг Markdown
- Встраивание YouTube видео

## Correctness Properties

1. **Подсказка скрывается после просмотра** - For any пользователя и подсказки, после закрытия подсказка не отображается повторно
2. **Шаг отмечается только один раз** - For any пользователя и шага, повторное выполнение не дублирует шаг
3. **Все шаги завершены = онбординг завершен** - For any пользователя, при выполнении всех 7 шагов is_completed = true
4. **Подсказка отображается только при выполнении условий** - For any подсказки, она показывается только если выполнены условия
5. **Прогресс-бар отображает корректный процент** - For any пользователя, процент = (выполненные шаги / 7) * 100
