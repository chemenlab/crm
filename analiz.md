  КРИТИЧЕСКИЕ УЯЗВИМОСТИ БЕЗОПАСНОСТИ

  1. Утечка хеша пароля на клиент — HandleInertiaRequests.php:132 передаёт
  $request->user()->password в браузер через Inertia shared data
  2. Обход проверки webhook-подписи — PaymentService.php:379 возвращает true если webhook_secret
  не настроен. Злоумышленник может отправлять фиктивные платёжные уведомления
  3. Обход авторизации при удалении — AppointmentController.php:416 позволяет любому
  аутентифицированному пользователю удалять записи без владельца через orWhereNull('user_id')
  4. Нет Rate Limiting на аутентификации и публичных эндпоинтах бронирования
